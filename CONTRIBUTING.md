# Contributing

This repo holds skills I built because I hit the same friction twice in my own work. That's the bar for what goes in it.

## Before opening a PR

- The skill should solve a real, repeatable problem — not a hypothetical one.
- Follow the existing `SKILL.md` structure (see any folder under `skills/` for the pattern).
- Tag provenance in the frontmatter (see the Provenance section in [README.md](README.md)). If you're adapting someone else's public work, credit them explicitly and confirm you have permission to redistribute it.
- Keep content in English.
- If the skill has an entry point registered elsewhere (e.g. `.claude-plugin/marketplace.json`), update it in the same PR.

## Smaller fixes

Typos, broken links, unclear instructions — just open a PR directly, no need to file an issue first.

## Proposing a new skill

Open an issue first with what problem it solves and why it doesn't already fit an existing skill. Saves both of us from a PR that gets rejected on scope alone.
